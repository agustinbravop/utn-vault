Cloud computing es la disponibilidad a demanda de recursos de hardware sin la administración activa del usuario. La ventaja está en que tenemos más tiempo para desplegar aplicaciones en lugar de administrar infraestructura. Existen múltiples proveedores (*vendors*) de nube, varios de estos con capas gratuitas.

Beneficios:

- Provisioning.
- Scalability.
- Expense.
- Robustness.
- Flexibility.
- Compatibility.
- Management.
- Ease of access.
- Resilience.
- Sturdy.

Ventajas:

1. On-demand self-service.
2. Broad network access.
3. Resource pooling.
4. Rapid elasticity.
5. Measured service.

La nube facilita el cumplimiento de requerimientos no funcionales.

Modelos de servicio:

1. **Infraestructure as a Service**.
2. **Platform as a Service**.
3. **Service as a Service**.
4. Otros: Function as a Service, Kubernetes as a Service, etc.

Modelos de despliegue:

- **Nube pública**: fácil de acceder pero poco control.
- **Nube privada**: una nube usada exclusivamente por una empresa.
- **Nube híbrida**: es la combinación de ambos enfoques. Requiere integración.

Arquitectura de cloud computing:

1. **Service layer**: portales web y servicios.
2. **Management platform layer**: asegurar estabilidad, seguridad, confiabilidad, eficiencia.
3. **Resource pool layer**: virtualización de la infraestructura.
4. **Infrastructure layer**: dispositivos de computing, network, storage, y seguridad.

Cloud computing soporta el desarrollo de nuevas tecnologías (AI, IoT, 5G, etc). Tendencias actuales: cloud native, edge computing, cloud-edge collaboration.
